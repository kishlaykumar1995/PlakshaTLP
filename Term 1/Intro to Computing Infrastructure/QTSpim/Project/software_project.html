[MIPS32 Assembly programming]  In this project, you will write a new MIPS32 assembly language program and simulate it on the MIPS simulator called QtSpim.  MIPS is the name of a simple RISC ISA.  Before attempting to do this question, familiarize yourself with the MIPS ISA and how to use the software by visiting the class website and reading all the documents on it related to MIPS and QtSpim.

Write a MIPS32 assembly language programs that takes a single 32-bit unsigned positive integer A, computes the number of 1's that are in the binary representation of that integer, and prints out the answer on the screen.  For example, if A=13 the answer should be 3. This is because 13 in binary is 1101 (ignoring leading zeros), so there are 3 1's in the binary representation; if A=31 the answer should be 5 (11111).  In writing the program, do not attempt to call any library function for parity or any other library function. The MIPS assembly code you write should do all of the computation by itself. Please put your entire code in the _main function.

Write your code by using the provided MIPS32 sample assembly language programs as a guide. Run all of those programs to verify that they work. Carefully read and understand at least two of the simpler examples as needed to write your code.

The computation should be done using bit masking. Recall that bitmasking is where we use bitwise operators to extract information about specific bits in a given number. For example, if A=101 is our number and M=001 is our mask, A & M will tell us whether the least significant bit of A is 1 or 0. For our program, our mask M will remain the same (1), and we will repeatedly right shift A and compute A & M. On our first iteration 101 & M = 1, on our second iteration 010 & M = 0, and on our third iteration 001 & M = 1. If we keep track, there were 2 1's in A. Follow the same logic in your program. Remember we are dealing with 32 bit numbers, not 3 bit numbers.

Your program should not take any input from the screen.  Instead, it should hard-code in the value of A in the program.  Do this by declaring a global variable A in your program, and initializing it to its known constant value, such as 13.  (To see how to declare an initialized global, look at example1.s among the sample MIPS programs. To do I/O, look at example3io.s.)

Example: If A = 13, then your MIPS session console output should look like the following:

There are 3 1's in the inputted number

When debugging your program, you may find it helpful to use QtSpim to step through the program instruction by instruction, examine the values of registers during execution and set breakpoints.  These are very easy to do -- use the QtSpim online help to learn to do these tasks and try them out for your program.
