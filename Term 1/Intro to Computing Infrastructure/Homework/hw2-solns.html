Q. 3.  Suppose you have a 16-bit machine with a 256-entry direct-mapped cache.  Each cache entry can hold 16 bytes of data.  A program is run on this machine that makes the following memory accesses in hexadecimal (each memory address addresses a byte of data):

            6F8E, 4F8E, 4F8D, 4FC2, 0000, 0002, 2F01, 4F8E, 2F67, 2E38, 3F69, 2E32

(3 + 5 + 3 + 3 = 14 points)

a.  How many bits wide is the cache's tag field?

Tag field + line field + line offset = 16.
ï¿½ Tag field + log2 256 + log2 16 = 16.
ï¿½ Tag field + 8 + 4 = 16.
ï¿½ Tag field = 16 - 12 = 4 bits.

b.  Assuming the cache is initially empty, what are the final contents of the cache after the program runs?  Draw a diagram of the cache showing its valid entries only, and all its fields for each entry, but fill in the tag fields only!

According to part (a), the first hex digit in the incoming address is the tag, the 2nd and 3rd hex digits together are the line field (which is the entry number), and the 4th hex digit is the offset field.

Thus, in the figure, cache entry F8 should have tag 4, FC should have tag 4, 00 should have tag 0, F0 should have tag 2, F6 should have tag 3, and E3 should have tag 2. All other entries should be empty.

c.  Suppose a cache hit takes 1 cycle, a cache miss takes 30 cycles, and a memory access without a cache takes 30 cycles.  How much time is saved by this cache on this program over an equivalent cache-less machine running this program?

In this sequence, a cache miss happens when a certain pair of tag and line fields are accessed for the first time. Otherwise, a cache hit occurs. In the sequence below, we show hits and misses by (h) and (m), respectively.

6F8E (m), 4F8E(m), 4F8D(h), 4FC2(m), 0000(m), 0002(h), 2F01(m), 4F8E(h), 2F67(m), 2E38(m), 3F69(m), 2E32(h)

So we have a total of 4 hits and 8 misses.

Time for memory accesses with cache: 4*1 + 8*30 = 244 cycles.
Time for memory accesses without cache: 12*30 = 360 cycles.
Therefore time saving is 360 - 244 = 116 cycles.

d.  Give an example each of spatial and temporal memory access locality exhibited by this program.

Spatial locality: 4F8E is followed by 4F8D.  Other examples: 0000 followed by 0002, or 2E38 followed by 2E32.
Temporal locality: 4F8E is followed by 4F8E.
