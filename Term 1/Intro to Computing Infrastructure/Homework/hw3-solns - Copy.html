Q. 1. A disk has an average seek time of 3ms (milliseconds), and a rotational speed of 7200 rotations per minute.  Each track on the disk has 32 sectors, each of 16KB, and the peak transfer from disk to main memory is equals the rate at which contiguous data is read off the disk. (Note KB = Kilobytes, whereas Kb = Kilobits).

(3 + 4 + 4 = 11 points)

a.     What is the peak transfer rate of this disk?

Rotation time = 1/Rotations per second = 1/ (7200/60) = 1/120 secï¿½ = 0.00833 seconds = 8.33ms

Peak transfer rate in MB/s = Size of one track in MB / Time for one rotation in seconds

= ((32*16) / 1024)/ (1/120) MB/seconds

= ï¿½ * 120 MB/sec = 60 MB/sec.

b.    What is the access time of this disk for reading one complete sector?

Average access time = average seek time + average rotational latency + transfer time

Average rotational latency = ï¿½ * (rotation time) = ï¿½ * (8.33) = 4.17ms.

Time to transfer one sector (sec) = Size of one sector in MB / Peak transfer rate in MB/sec

=  (16 / 1024)/ 60 seconds = 0.00026 seconds.

= 0.26 ms.

⇒Average access time for one sector = 3ms + 4.17ms + 0.26ms = 7.43ms.

c.     A file of size 4MB is loaded from this disk. What is the total access time for this file? Assume that files are stored non-contiguously, and each contiguous portion making up the file is 3 sectors long on average. Keep in mind that when switching from one contiguous portion to another, a new seek time and rotational latency is incurred.

Time to read each contiguous portion = Avg. seek time + avg. rot. latency + time to read 3 contiguous sectors

= 3ms + 4.17ms + 3*0.26ms

= 7.95ms

 

Number of contiguous portions = size of file in KB/length of 3 sectors in KB

= 4*1024/(3*16)

= 86 (rounding up to nearest integer)

 

Therefore, total time to read file in seconds = # of contiguous portions * Time to read each contiguous portion in seconds

= 86 * 0.00795 seconds

= 0.684 seconds

 

Q. 2. A machine supports 16 bit instructions and has 32 general purpose registers. The machine supports the following types of instructions:

(3 + 8 + 5 + 4 = 20 points)

ï¿½         60 instructions with two register fields            (Type A)

ï¿½         30 instructions with one register field             (Type B)

ï¿½         3 instructions with a 10 bit immediate field    (Type C)

ï¿½         26 instructions with no operands                    (Type D)

(a)  Is it possible to design an instruction word format for this machine without expanding opcodes?  If so, give a design.  If not, give a reason why such a design is not possible.  (A design without expanding opcodes is where the entire opcode for all instructions is in a single length-invariant opcode field.)

No, a design without expanding opcodes is not possible.  The reason is that with a single length-invariant opcode a total of 60+30+3+26= 119 instructions requires ceiling(log2 119) = 7 bits of opcode.  This leaves 9 bits for operands.  Instruction type 1 needs two register fields of log2 32 = 5 bits each, for a total of 10 bits.  Thus 9 bits are not enough to encode type 1 instructions. Similarly, type 3 instructions cannot be encoded.

(b)  Design an instruction word format for this machine using expanding opcodes.  For each of the four types of instructions, draw the 16-bit instruction; divide it among its components; label each with name and number of bits; and specify the particular opcode combination numbers for each. (The choice of combinations to use is yours).

In the solution below, the first six bits are the first-level opcode; the next five bits are the second-level opcode when present; and the last five bits are the third-level opcode when present.

Instruction type A:
| 6 bit opcode | 5 bit reg field | 5 bit reg field |
opcodes range from 000000 to 111011

Instruction type B:
| 111100 | 5 bit opcode | 5 bit reg field |
opcodes range from 00000 to 11101

Instruction type C:
| 6 bit opcode | 10 bit immediate field |
opcodes range from 111101 to 111111

Instruction type D:
| 111100 | 11110 | 5 bit opcode |
opcodes range from 00000 to 11001

(c)  How many more instructions with no operands (Type D) could this machine support? Be careful to consider unused combinations in all opcode levels.

In type D instructions, there are 32 combinations available in the third-level opcode of which only 26 are used, so 6 are available for new instructions of type D.  Further, let us consider how many unused combinations remain in the other opcodes.  The first-level opcode's 6 bits are exhausted.  In the second-level opcode, there is one unused combination, 11111.  By using this combination for type D instructions as well, by varying the third-level opcode, and additional 32 type D instructions are possible.  This results in a total of 6 + 32 = 38 new type D instructions possible.

(d)  Suppose you need the machine to support one instruction with three register fields. Since there is not enough room in the instruction word for this new instruction, you must reduce the number of instructions currently supported. Suppose you can only afford to reduce the number of instructions of one particular type. Which type must you choose and what are the new (reduced) number of instructions of that type?

We need:
| 0 | 5 bit reg field | 5 bit reg field | 5 bit reg field |
The only type of instruction that can be reduced to support this is type A. It becomes:
| 1 | 5 bit opcode | 5 bit reg field | 5 bit reg field |
Since an entire bit of A's opcode field must be sacrificed, this means that there can only be 32 type A instructions maximum instead of 64. And since we still need 4 combinations from type A to support B, C, and D, that means we can only have 28 type A instructions. This means a reduction of 60-28 = 32 type A instructions.

 

Q. 3. A possible reduced instruction set machine might support the following instructions:

(8 + 3 + 2 + 2 = 15 points)

ï¿½         ADD rd, rs, rt

ï¿½         NAND rd, rs, rt

ï¿½         LUI rd, imm

ï¿½         BEQ rs, rt, addr

ï¿½         LW rd, (rt)

ï¿½         SW rs, (rt)

ï¿½         LLI rd, imm

ï¿½         JALR rd, rs

Suppose that this machine has 8 general purpose registers. Also suppose that the immediate operand fields on LUI and LLI can hold any integer from -128 to 127. The width of the opcode field is the same for every instruction.

LUI (load upper immediate) and LLI (load lower immediate) allow an immediate number to be loaded into a register. LUI loads the immediate number into the exact upper half of the register specified by rd while LLI loads the immediate number into the exact lower half of rd. Assume that no instructions beside LUI and LLI are needed to get an immediate number into a register, and assume that the immediate field is the same size in LLI and LUI.

(a)  Ignoring BEQ, how many bits are needed in the instruction word to support all of these instructions?

ADD needs 3+3+3+3=12 bits
NAND needs 3+3+3+3=12 bits
LUI needs 3+3+8=14 bits
BEQ needs 3+3+3+addr bits
LW needs 3+3+3=9 bits
SW needs 3+3+3=9 bits
LLI needs 3+3+8=14 bits
JALR needs 3+3+3=9 bits
14 bits will support all the instructions.

(b)  If the instruction word is to contain as few bits as possible, how many bits is the BEQ instruction's address field? Assuming that BEQ uses relative addressing, how many instructions ahead can it branch to? (Consider that BEQ can branch backwards as well as forwards and assume that the BEQ PC-relative displacement value is the number of instructions of displacement.).

3+3+3+addr = 14 bits
Maximum addr width is 5 bits. 5 bits can indicate 32 integers, or a number from -16 to 15. This means that BEQ can branch backwards by 16 instructions, or forward by 15.

(c)  How many bits are stored in each general purpose register?

Since the immediate fields are 8 bits, the register size must be 8*2 or 16 bits (LUI loads the upper 8 bits, LLI loads the lower 8 bits).

(d)  If LW and SW use register indirect addressing, how many addresses can this machine index?

A register has 16 bits, so it can address 216 = 64k addresses

 

Q4.   In order to perform a disk or network access, it is typically necessary for a user to have the operating system communicate with the disk or network controllers.  Suppose that in a particular 500 MHz computer, it takes 1,000 cycles to trap (interrupt) to the OS.  In addition, it takes 8ms for the OS to perform a disk access, or 4ms for the OS to perform a network access.

(8 + 4 = 12 points)

 

(a)  In a disk access, what percentage of the delay time is spent in trapping to the OS?  How about in a network access?

 

        Time for trap = 1,000/500,000,000 seconds = 2 ms.

Disk access total time: 2ms + 8 ms = 8.002 ms
% delay trapping to OS= 0.002/8.002 = 0.02%
Network access total time: 2ms + 4 ms = 6 ms
% delay trapping to OS= 2/6 = 33.3%

  

(b)  Suppose that we can somehow reduce the time for the OS to communicate with the network by 40%.  The trap time is unchanged. By what percentage can we reduce the total time for a network access?

Fraction of original time = (2 ms + 4 ms * 0.6)/(2 ms + 4 ms) = 4.4/6 =  73.3% of original time (26.7% reduction)

 

 

Q5.  A 32-bit DLX computer transfers a 512-byte sector (128 words) from the hard disk to the main memory by using DMA.  The disk reads the sector from the platter into a special memory called the disk cache which is part of the disk controller.  Thereafter the data is transmitted from the disk cache to the main memory via the bus by using DMA.  Compute this DMA transfer time from disk cache to main memory.  The following are the parameters we know:

ï¿½       The bus is 4 words wide and the time to propagate a signal on the bus is 4 processor cycles. (This is often called the bus cycle time.)

ï¿½       The disk cache is 1 word wide and has an access time of 1 processor cycle.

ï¿½       The main memory is 16 words wide and has an access time of 80 processor cycles. Assume that main memory accessed cannot be pipelined (i.e., we must wait for the previous access to finish before the next access can be issued.

The process of transfer is as follows.  First, four words are read from the disk cache into hardware registers.  When done, they are sent together on the bus in one transfer.  When four successive bus transfers are received at the memory, they are written together to the memory.  Different stages in this process are overlapped (pipelined) to the extent possible -- carefully think through what can be overlapped. 

 

(12 points)

    # of main memory writes = 128/16 = 8 writes.

    Main memory write time = 80 cycles.

    Consider what happens for first write:

    Time to read four words from disk cache = 4 cycles.

    Bus transfer time = 4 cycles.

   ï¿½ Time for first 4 words to arrive at main memory = 8 cycles.

    Thereafter, four new words arrive every four cycles (because bus transfer and disk cache time are overlapped).

    Time for remaining 3 bus transfers per write = 3 * 4 = 12 cycles.

   ï¿½ total time for first write = Time for first transfer + time for remaining 3 transfers + time for write

         = 8 + 12 + 80 = 100 cycles.

    For remaining writes:

    Time for remaining seven writes is just 80 cycles each as transfer time is completely overlapped by earlier write.

    ï¿½ Total DMA time = Time for first write + 7* Time for each remaining writes

         = 100 + 7*80 = 660 cycles.

 

Q6.    Consider the identical computer to the one in question 2, but now the sectors are transferred from disk cache to main memory by using processor-involved software transfers, instead of DMA.  The transfers are done one word at a time by a loop in assembly of the form:

                           

                             _loop: LW  ...                    /* from memory-mapped disk cache address */

                                        SW  ...                    /* to main memory */

                                        ADDI ...                 /* register increment for address register to main memory */

                                        BNEZ ..., _Loop     /* branch back to LW for next iteration for next word */

 

Memory mapped locations (the loads above) are not cached by the processor caches.  Further, the processor caches use write through for main memory locations. (In a write-through cache, all stores are written to main memory immediately instead of waiting for eviction of that data.). Hence both the loads and stores use the bus. The branch predictor is perfect, and thus there are no stalls from control hazards.   What is the total transfer time in software for the sector?  How many times faster is DMA in question 2 compared to this software transfer?

 

(10 points)

    Time for LW = 1 cycle for pipeline + 4 cycles for bus for request + 1 cycle for disk cache read

                                    + 4 cycles for bus for data

                          = 10 cycles.

    Time for SW = 1 cycle for pipeline + 4 cycles for bus for data + 80 cycles for main memory write

                          = 85 cycles.

    Time for ADDI and BNEZ = 2 cycles together.

   ï¿½ Time per iteration = 10 + 85 + 2 =97 cycles.

   ï¿½ Total time for transferring sector of 128 words = 128 * 97 = 12416 cycles.

   ï¿½ DMA is 12416/660 = 18.81 times faster.
